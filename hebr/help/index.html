List of character apps

Updated 26-Aug-2020 • tags apps, pickers, hebrew

This Unicode character app allows you to produce or analyse runs of Hebrew text. character apps are especially useful for people who don't know a script well, as characters are displayed in ways that aid identification.

If something is broken or missing raise an issue. For version information see the Github commit list. The pictures of the UI used in this page are taken from a variety of character apps.

About the selection area

Includes characters in the Unicode Hebrew block.

Controls above the text area

Transliterate. Converts the contents of the text area to a Latin transliteration that provides a one-to-one correspondence between Hebrew and Latin symbols.

The transliteration scheme was developed for this character app, and, where possible, uses symbols that represent sounds typically associated with their Hebrew equivalents, rather than the sometimes fairly arbitrary selection of diacritics found in other systems. Of course, the complexities of mapping graphemes to sounds means that this is far from a real phonetic transcription, and it doesn't show inherent vowels, but the process is reversible (unlike most transcriptions).

Transliterations produced this way are particularly useful for understanding the composition of text in scripts, where the individual parts are not well separated in the text.

Make vocab. If you want to create data for the vocabulary app, you need to make a list of 2-5 items, separated by | in the following order:

[1] native text, [2] meaning, [3] basic transcription (optional), [4] alternative transcription (optional), [5] notes (optional)

Hint: For RTL script character apps it often helps to set the direction of the text area to LTR. (To do this, click on the →︎ arrow below the bottom right of the text area.)

Normally, the native text you supply should be pointed. Clicking on Make vocab will automatically convert the native text to unpointed (except for dagesh and shin/sin dots), and add the pointed version as the alternative transcription item.

The basic transcription should be in IPA, wherever possible. You can also add different transcription, such as an ISO or LOC transcription, to this field, surrounded with parentheses. This will be styled differently from the IPA.

You can separate items using / rather than |. Clicking on Make vocab will convert all / characters to |. (It will also produce some markup in the secondary text area.)

Here's a Hebrew example showing what was typed in.

פִּינְגְּוִין/penguin/ˈpinɡwin
גְּמָלִים/camels/ɡmaˈlim//plural

And here's the result after clicking on Make vocab after highlighting each line.

פּינגּוין|penguin|ˈpinɡwin|פִּינְגְּוִין
גּמלים|camels|ɡmaˈlim|גְּמָלִים|plural

If you have more than one line of text in the text area, you must highlight the row you want to convert.

Vocab to Markup. If you have a line of vocab data in the text area (see above), this will produce markup that includes all the items on the line except for the notes. That markup can then be dropped into an HTML page.

For example, this vocab data line:

פּינגּוין|penguin|ˈpinɡwin|פִּינְגְּוִין

will produce this markup:

<span class="charExample" translate="no"><bdi class="ex" lang="he" dir="rtl">פּינגּוין</bdi> <bdi class="trans">fˑyngˑv̇yn̽</bdi> (<bdi lang="he">פִּינְגְּוִין</bdi>) <bdi class="ipa">ˈpinɡwin</bdi> <bdi class="meaning">penguin</bdi></span>
