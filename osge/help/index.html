This Unicode character app allows you to produce or analyse runs of Osage text. character apps are especially useful for people who don't know a script well, as characters are displayed in ways that aid identification.

If something is broken or missing raise an issue. For version information see the Github commit list. The pictures of the UI used in this page are taken from a variety of character apps.

About the selection area

Includes characters in the Unicode Osage block and the Combining Diacritical Marks block that are used for writing text in the Osage script. The characters in the chart are arranged along phonetic lines. When you mouse over the shapes, the character app displays the joining forms just above.

Controls above the text area

Transliterate. Converts the contents of the text area to a Latin transliteration that provides a one-to-one correspondence between Osage and Latin symbols.

The transliteration scheme was developed for this character app, and, where possible, uses symbols that represent sounds typically associated with their Osage equivalents, rather than the sometimes fairly arbitrary selection of diacritics found in other systems. Of course, the complexities of mapping graphemes to sounds means that this is far from a real phonetic transcription, but the process is reversible (unlike most transcriptions).

Make vocab. This helps speed up the creation of entries for the vocabulary app. You can create a vocab item with up to 3 parts, delimited by |, in the following order: [1] native text, [2] meaning, [3] notes (optional). Much of the time, only the first 2 items are needed.

The text will be replaced with the following:

native_text|meaning|transcription[|notes]

Upper-/Lower-case. Converts text to upper- or lowercase.

Transcribe to IPA. Converts Osage text to an IPA transcription. This is only a rough, phonemic transcription, and the result should be checked before use.

Vocab to Example. If you have a line of vocab data, like those shown above, this will add example markup around the term.

Input aids

ISO to Osage. Allows you to generate Osage text from an ISO 9985 transcription. It works as described in the section just above about Reverse transliteration, but uses a different panel to set up the transcription type-in mode.
